# Codex Planning Guide

Codex should keep plans concise, step-based, and easy to resume.

## Planning Rules

- use short step-based plans
- distinguish clearly between analysis, implementation, and follow-up work
- keep plans practical and low-overhead
- avoid overengineering
- prefer the smallest effective plan for the task
- update plans when scope changes

## Plan Structure

When useful, structure plans with these categories:

1. Analysis
2. Implementation
3. Validation
4. Follow-up

## Task Start Format

```md
Task:

Relevant Files:

Current Understanding:

Plan:
1.
2.
3.

Execution Mode:
- Assistant or Primary Engineer

CLI Needed:
- Yes / No (why)

Next Step:
```

## Resume-Friendly Planning

Plans should make it easy to resume work after interruption.

Include:
- the current task
- relevant files
- current state
- next action
- major risks or blockers

## Execution Guidance

- analyze first when context is unclear
- implement only after the path is reasonably clear
- validate important changes
- record follow-up items separately from the immediate task
